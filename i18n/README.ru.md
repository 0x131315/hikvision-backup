## Быстрый старт

### Запуск предварительно собранного бинарного файла
1. Загрузите [последнюю версию](https://github.com/0x131315/hikvision-backup/releases/latest) для вашей платформы.

2. Распакуйте архив и откройте папку:
```bash
cd <папка проекта>
```
3. Создайте файл конфигурации:
```bash
cp .env.dist .env
```
4. Отредактируйте `.env`, затем запустите:
```bash
./hikvision-backup
```

### Типичные сценарии автоматизации
Вы можете запустить этот инструмент из `cron`, `systemd` или другого планировщика задач.

## Конфигурация

Приложение считывает настройки из файла `.env` и из переменных окружения.
Переменные окружения имеют более высокий приоритет, чем `.env`.

| Переменная | Необходимый | По умолчанию | Пример | Описание |
|---|---|---|---|---|
| `DOWNLOAD_DIR` | Yes | - | `/home/user/camera_videos` | Локальная папка для загруженных видео. |
| `CAM_HOST` | Yes | - | `192.168.1.10`, `https://cam.local:8443` | Адрес камеры или полный URL-адрес. Если схема не задана, используется `https://`. |
| `CAM_USER` | Yes | - | `admin` | Имя пользователя камеры |
| `CAM_PASS` | No | empty | `secret` | пароль камеры |
| `CAM_INSECURE_SKIP_VERIFY` | No | `false` | `true` | Пропустить проверку TLS-сертификата. Использовать только в доверенных сетях. |
| `SCAN_LAST_DAYS` | No | `0` | `3` | Сканируйте видео только за последние N дней. `0` означает отсутствие ограничений. |
| `SCAN_FROM_LOCAL_LATEST` | No | `0` | `2` | Найдите последние местные видеозаписи и выполните сканирование начиная с указанной даты за вычетом N дней. |
| `HTTP_RETRY_CNT` | No | `3` | `5` | Количество повторных попыток для ошибок запроса (`5xx`, `401`, `403`) |
| `HTTP_TIMEOUT` | No | `120` | `30` | Таймаут HTTP в секундах. `0` означает отсутствие ограничения по времени. |
| `NO_PROXY` | No | `false` | `true` | Игнорировать настройки прокси-сервера из переменных среды. |

## Справочник по командной строке

| Вариант | Уровень лога | Вариант использования |
|---|---|---|
| _(без флагов)_ | Info | Обычный ежедневный пробег |
| `--verbose`, `-vv` | Debug | Отладка поведения приложения |
| `--verbose-http`, `-vvv` | Отладка + трассировка HTTP | Отладка запросов к API камеры |
| `-v`, `--version` | Без бега | Показать информацию о версии и выйти |

Шаблоны использования:
```bash
./hikvision-backup
./hikvision-backup --verbose
./hikvision-backup --verbose-http
./hikvision-backup --version
```

## Как это работает

1. Запросите список файлов у API камеры (ISAPI) для заданного временного диапазона.

2. Для каждого видео используйте время начала в качестве имени файла.

3. Используйте размер файла камеры в качестве ожидаемого размера.

4. Проверьте, существует ли файл в `DOWNLOAD_DIR`.

5. Если файл отсутствует, загрузите его.

6. Если файл слишком мал, удалите его и загрузите снова.

7. Повторите неудачные HTTP-запросы до `HTTP_RETRY_CNT`.

8. После загрузки сравните размер файлов и удалите поврежденные файлы.

## Сборка из исходного кода

1. [Установите Go](https://go.dev/doc/install).

2. Клонируйте проект и выполните сборку:
```bash
git clone https://github.com/0x131315/hikvision-backup.git <project-folder>
cd <project-folder>
make build
```
3. Создайте конфигурационный файл и запустите:
```bash
cp .env.dist .env
./hikvision-backup
```
