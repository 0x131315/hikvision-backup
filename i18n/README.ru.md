## Быстрый старт

### Запуск предварительно собранного бинарного файла
1. Загрузите [последнюю версию](https://github.com/0x131315/hikvision-backup/releases/latest) для вашей платформы.

2. Распакуйте архив и откройте папку:
```bash
cd <папка проекта>
```
3. Создайте файл конфигурации:
```bash
cp .env.dist .env
```
4. Отредактируйте `.env`, затем запустите:
```bash
./hikvision-backup
```

### Типичные сценарии автоматизации
Вы можете запустить этот инструмент из `cron`, `systemd` или другого планировщика задач.

## Конфигурация

Приложение считывает настройки в следующем порядке:
1. `.env`
2. переменные среды
3. параметры командной строки

Побеждает последний источник.

| Переменная | Опция командной строки | Необходимый | По умолчанию | Пример | Описание |
|---|---|---|---|---|---|
| `DOWNLOAD_DIR` | `--download-dir`, `-d` | Yes | - | `/home/user/camera_videos` | Локальная папка для загруженных видео. |
| `CAM_HOST` | `--cam-host`, `-H` | Yes | - | `192.168.1.10`, `https://cam.local:8443` | Адрес камеры или полный URL-адрес. Если схема не задана, используется `https://`. |
| `CAM_USER` | `--cam-user`, `-u` | Yes | - | `admin` | Имя пользователя камеры |
| `CAM_PASS` | `--cam-pass`, `-p` | No | empty | `secret` | пароль камеры |
| `CAM_INSECURE_SKIP_VERIFY` | `--cam-insecure-skip-verify`, `-k` | No | `false` | `true` | Пропустить проверку TLS-сертификата. Использовать только в доверенных сетях. |
| `SCAN_LAST_DAYS` | `--scan-last-days`, `-s` | No | `0` | `3` | Сканируйте видео только за последние N дней. `0` означает отсутствие ограничений. |
| `SCAN_FROM_LOCAL_LATEST` | `--scan-from-local-latest`, `-l` | No | `0` | `2` | Найдите последние местные видеозаписи и выполните сканирование начиная с указанной даты за вычетом N дней. |
| `HTTP_RETRY_CNT` | `--http-retry-cnt`, `-r` | No | `3` | `5` | Количество повторных попыток для ошибок запроса (`5xx`, `401`, `403`) |
| `HTTP_TIMEOUT` | `--http-timeout`, `-t` | No | `120` | `30` | Таймаут HTTP в секундах. `0` означает отсутствие ограничения по времени. |
| `NO_PROXY` | `--no-proxy`, `-P` | No | `false` | `true` | Игнорировать настройки прокси-сервера из переменных среды. |

## Справочник по командной строке

| Вариант | Уровень лога | Вариант использования |
|---|---|---|
| _(без флагов)_ | Info | Обычный ежедневный пробег |
| `--verbose` | Debug | Отладка поведения приложения |
| `--verbose-http` | Отладка + трассировка HTTP | Отладка запросов к API камеры |
| `-v`, `--version` | Без бега | Показать информацию о версии и выйти |

В параметрах конфигурации используются те же имена, что и ключи переменных окружения в формате kebab-case.
Устаревшие псевдонимы `-vv` и `-vvv` по-прежнему работают в режимах отладки.

Шаблоны использования:
```bash
./hikvision-backup
./hikvision-backup --verbose
./hikvision-backup --cam-host=192.168.1.10 --cam-user=admin --download-dir=/data/cam
./hikvision-backup -H 192.168.1.10 -u admin -d /data/cam -s 3
./hikvision-backup --verbose-http
./hikvision-backup --version
```

## Как это работает

1. Запросите список файлов у API камеры (ISAPI) для заданного временного диапазона.

2. Для каждого видео используйте время начала в качестве имени файла.

3. Используйте размер файла камеры в качестве ожидаемого размера.

4. Проверьте, существует ли файл в `DOWNLOAD_DIR`.

5. Если файл отсутствует, загрузите его.

6. Если файл слишком мал, удалите его и загрузите снова.

7. Повторите неудачные HTTP-запросы до `HTTP_RETRY_CNT`.

8. После загрузки сравните размер файлов и удалите поврежденные файлы.

## Сборка из исходного кода

1. [Установите Go](https://go.dev/doc/install).

2. Клонируйте проект и выполните сборку:
```bash
git clone https://github.com/0x131315/hikvision-backup.git <project-folder>
cd <project-folder>
make build
```
3. Создайте конфигурационный файл и запустите:
```bash
cp .env.dist .env
./hikvision-backup
```
